ID Validation Scripts (Buyer & Seller)

Benchmarked on 6 February 2026. Covers validate-buyer and validate-seller.

Dataset Size Processing Time Throughput
10,000 records ~200 ms ~50,000 rec/s
100,000 records ~2 seconds ~50,000 rec/s
250,000 records ~5 seconds ~50,000 rec/s
Analysis: Throughput is consistent at ~50K records/second across dataset sizes, indicating O(n) linear scaling. The system uses pre-compiled regex patterns and indexed lookups (Python dictionaries for O(1) access) for country codes and ID formats.

Scalability (1.5M Records)

Benchmarked on 17 February 2026 using scripts/benchmark_scalability.py.

Metric Value
Dataset size 1,500,000 records
Processing time 135.31 seconds
Throughput 11,086 records/second
Peak memory 4,062 MB
Average memory 3,912 MB
Memory per record 2.77 KB
Valid records 500,000 (33%)
Invalid records 1,000,000 (67%)
Corrected records 900,000 (60%)
Memory note: At 1.5M records the peak memory usage reaches ~4 GB. For machines with limited RAM, large datasets should be processed in chunks. Typical quarterly workloads (25K–50K records) use well under 500 MB.

Methodology

Benchmarking Tools

Script Purpose
scripts/benchmark_performance.py Multi-iteration benchmarks with tracemalloc and optional cProfile
scripts/benchmark_scalability.py Scalability testing across dataset sizes (100K to 1.5M)
scripts/benchmark_accuracy_testing.py Accuracy testing script-specific benchmarks
scripts/benchmark_accuracy_validation.py Validation logic micro-benchmarks
scripts/benchmark_chunked.py Chunked processing performance comparison
scripts/profile_performance.py Detailed profiling with function-level timing

Performance Techniques

  • Pre-compiled regex patterns: All 68 ID format patterns are compiled at import time
  • Singleton managers: CountryDataManager and IDFormatManager load data once
  • Dictionary-based lookups: O(1) access for country codes, incident codes, and ID format routing
  • Lazy loading: Reference data loaded on first access, not at module import
  • Batch processing: Config v2.0 supports processing multiple incidents in a single pipeline run

How to Run Benchmarks

# Standard performance benchmark
python scripts/benchmark_performance.py --iterations 3

# Scalability test
python scripts/benchmark_scalability.py --output benchmarks/scalability_results.json

# With profiling
python scripts/benchmark_performance.py --iterations 1 --profile

Recommendations

Scenario Recommendation
Typical quarterly workload (<50K records) No special consideration needed. Runs in under 1 second.
Large datasets (100K–500K) Monitor memory usage. Consider increasing Python heap if needed.
Very large datasets (>500K records) Use chunked processing or split input files. Monitor system memory.
Low-memory machines (<4 GB RAM) Keep datasets under 250K records per run.
