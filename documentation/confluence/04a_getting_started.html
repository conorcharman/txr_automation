Prerequisites

Requirement Version Notes
Python ≥ 3.10 Required for match statements and type union syntax
Conda (recommended) Any recent Miniconda or Anaconda. Alternatively use pip + venv
Git Any recent For cloning the repository
macOS / Windows / Linux — All platforms supported. macOS is the primary development platform

Environment Setup

Option A: Conda (Recommended)

# 1. Clone the repository
git clone <repository-url> txr_automation
cd txr_automation

# 2. Create the Conda environment from environment.yml
conda env create -f environment.yml

# 3. Activate the environment
conda activate txr_automation

# 4. Install the package in editable mode
pip install -e .

The environment.yml installs all dependencies:

name: txr_automation
channels:
  - conda-forge
dependencies:
  - python>=3.10
  - pyyaml>=6.0
  - pandas>=2.0.0
  - openpyxl>=3.1.0
  - pyside6>=6.6
  - pytest>=7.4.0
  - pytest-cov>=4.1.0
  - pip

Option B: pip + venv

# 1. Clone and enter the repository
git clone <repository-url> txr_automation
cd txr_automation

# 2. Create a virtual environment
python3 -m venv .venv
source .venv/bin/activate  # macOS/Linux
# .venv\Scripts\activate   # Windows

# 3. Install dependencies
pip install -r requirements.txt

# 4. Install the package in editable mode
pip install -e .

Verify Installation

# Check the package is installed
python -c "import accuracy_testing; print('accuracy_testing OK')"
python -c "import core; print('core OK')"

# Check console scripts are available
buyer-id-validation --help
replay-phase2 --help
txr-gui --help
If all three commands produce output without errors, you are ready to go.

Running Your First Script

Step 1: Create Local Configuration

# Copy the template configuration for buyer ID validation
mkdir -p config/local/accuracy_testing
cp config/templates/accuracy_testing/buyer_validation_template.yaml \
   config/local/accuracy_testing/buyer_validation.yaml

Edit the copied file to set your input/output paths:

# config/local/accuracy_testing/buyer_validation.yaml

mode: single    # 'single' for one file, 'batch' for directory scan

paths:
  input_file: "data/test/buyer_id_sample.csv"
  output_file: "data/output/buyer_id_results.csv"
  log_output: "logs"

processing:
  incident_code: "7_5"
  log_level: "INFO"

Step 2: Run the Validation

# Using the console script (registered in setup.py)
buyer-id-validation --config config/local/accuracy_testing/buyer_validation.yaml

# Or using Python module syntax
python -m src.accuracy_testing.scripts.buyer_id_validation \
    --config config/local/accuracy_testing/buyer_validation.yaml

Step 3: Review Output

The script produces:

  • Full results CSV — all records with validation columns added (Error, Error_Details, Correction, Match, etc.)
  • Errors-only CSV — filtered to only records flagged as errors (filename suffixed _errors)
  • Log file — structured JSON log in the configured log directory
  • Console summary — statistics printed to stdout (total records, errors found, processing time)

Other Common Scripts

# Seller ID validation
seller-id-validation --config config/local/accuracy_testing/seller_validation.yaml

# Decision maker validation (buyer side)
validate-ftbdm --config config/local/accuracy_testing/ftbdm_validation.yaml

# SQL extract generation (batch mode)
sql-extract --config config/local/accuracy_testing/sql_extract.yaml

# Replay Phase 2 processing
replay-phase2 --config config/local/replay/phase2.yaml

# Run all validations at once
run-all-validations --config config/local/accuracy_testing/run_all_validations.yaml

Configuration

Config templates → local copies. Templates in config/templates/ are version-controlled. Copy them to config/local/ (gitignored) and customise. See 4c. Configuration System for full details.

Environment Variables

All configuration values can be overridden with TXR_-prefixed environment variables:

export TXR_LOG_LEVEL=DEBUG
export TXR_INPUT_FILE=/path/to/data.csv
export TXR_OUTPUT_DIR=/path/to/output/

Precedence (highest to lowest):

  1. Command-line arguments (--config, --log-level, etc.)
  2. Environment variables (TXR_*)
  3. Local YAML configuration file
  4. Default values in code

Running Tests

# Run the full test suite
pytest

# Run with verbose output
pytest -v

# Run a specific test module
pytest tests/test_accuracy_testing/test_buyer_id_validation.py

# Run tests with coverage report
pytest --cov=src --cov-report=html

# Run only fast unit tests (exclude integration)
pytest -m "not integration"
Current status: 466 tests, 100% pass rate. Expected run time: ~30 seconds.

Launching the GUI

# Using the console script
txr-gui

# Or using Python module syntax
python -m src.gui.main

The GUI provides a tabbed interface for:

  • Accuracy Tab — run any of the 9 accuracy testing validations with file/config pickers
  • Replay Tab — Phase 2/3 replay processing
  • FIRDS Tab — refresh FIRDS cache and check instrument reportability
  • GLEIF Tab — refresh LEI cache and perform entity lookups
  • Settings Tab — configure paths and preferences
