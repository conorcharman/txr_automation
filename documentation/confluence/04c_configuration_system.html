Configuration System

All scripts support a unified configuration cascade: CLI arguments > environment variables > YAML file > defaults. YAML templates are version-controlled in config/templates/; local copies live in config/local/ (gitignored).

Configuration Cascade

Every CLI script resolves configuration values using the following priority order (highest wins):

PrioritySourceExample
1 (highest)CLI arguments--log-level DEBUG, --dry-run
2Environment variablesTXR_ACCURACY_PROCESSOR_LOG_LEVEL=DEBUG
3YAML configuration file--config config/local/accuracy_testing/buyer_validation.yaml
4 (lowest)Hard-coded defaultslog_level: INFO, batch_size: 1000

Resolution Flow

CLI script invoked
    |
    +-- --config PATH provided?
    |       Yes --> Load YAML file
    |       No  --> --use-env flag?
    |                   Yes --> Load from TXR_* environment variables
    |                   No  --> Auto-discover config/local/[category]/[script].yaml
    |                           Not found --> Exit with usage guidance
    |
    +-- Override with any explicit CLI arguments (--log-level, --dry-run, etc.)
    |
    +-- Fill remaining gaps with hard-coded defaults

Local Configuration Setup

Initial Setup

  1. Copy a template to the local directory: 
    cp config/templates/accuracy_testing/buyer_validation_template.yaml \
   config/local/accuracy_testing/buyer_validation.yaml
  2. Edit the local copy with your file paths and preferences
  3. Local files are automatically gitignored — they will not be committed
Never commit local config files. The config/local/ directory is gitignored to prevent accidental exposure of environment-specific paths. Always use templates from config/templates/ as the starting point.

Gitignore Pattern

# From .gitignore
config/local/

Config Manager API

Two config manager classes handle configuration loading and validation.

AccuracyConfigManager

Location: src/accuracy_testing/processor.py

MethodSignatureDescription
load_from_yaml (config_path: str) → Dict Parse a YAML file and return the configuration dictionary
load_from_env (prefix: str = "TXR_ACCURACY_") → Dict Build configuration from environment variables with the given prefix
get_path_config (config: Dict) → AccuracyPathConfig Extract and validate paths into an AccuracyPathConfig dataclass
get_processor_config (config: Dict) → AccuracyProcessorConfig Extract processor settings into an AccuracyProcessorConfig dataclass

ConfigManager (Core)

Location: src/core/config/config_manager.py

MethodSignatureDescription
load_from_yaml (config_path: str, use_cache: bool = True) → Dict Parse a YAML file (with optional caching for repeated loads)
load_from_env (prefix: str = "TXR_") → Dict Build configuration from TXR_-prefixed environment variables

Configuration Dataclasses

DataclassModuleKey Fields
AccuracyPathConfig accuracy_testing.processor input_file, template_file, output_file, log_output, italian_tracker, main_tracker
AccuracyProcessorConfig accuracy_testing.processor log_level, verbose, batch_size

ProcessorConfig core.config log_level, verbose, batch_size

Buyer Validation — Batch Mode

mode: "batch"

testing_period:
  fiscal_year: "FY26"
  quarter: "Q1"

batch:
  incidents: "auto"
  auto_incidents:
    - "7_35"
    - "7_37"
    - "7_39"

  paths:
    extract_dir: "data/extracts/FY26/Q1"
    template_dir: "data/templates/FY26/Q1"
    output_dir: "data/validated/FY26/Q1"
    log_output: "logs"

  filename_patterns:
    extract: "{incident}_{fiscal_year}_{quarter}.csv"
    template: "{fiscal_year} {quarter} {incident}.csv"

processor:
  log_level: "INFO"
  verbose: false

options:
  dry_run: false
  show_progress: true

Troubleshooting

SymptomCauseResolution
Script exits with “No configuration found” No --config or --use-env specified and no auto-discoverable file exists Copy a template to config/local/ and pass --config PATH
FileNotFoundError for input/template files Paths in YAML are relative and the working directory differs from expected Use absolute paths in YAML, or always run scripts from the project root
Environment variables not picked up Missing --use-env flag, or incorrect prefix (TXR_ACCURACY_ vs TXR_) Ensure the flag is passed and the variable prefix matches the script category
Batch mode finds no files Filename patterns do not match actual filenames on disc Check filename_patterns against actual files; verify testing_period values
YAML parsing error Invalid YAML syntax (indentation, special characters) Validate YAML with a linter; ensure strings with colons or special characters are quoted
