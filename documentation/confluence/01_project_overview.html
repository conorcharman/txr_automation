Executive Summary

TXR Automation (Phase 6) is a modern full-stack web application for financial transaction reporting validation. The system provides an interactive React 19 frontend with real-time validation feedback, a FastAPI backend with async job processing, and a powerful Python validation engine. It handles buyer/seller identification validation, decision maker validation, pricing checks, and data reconciliation across 68 country-specific ID format patterns.

Status: Phase 6 (Integration & Testing) in progress. The VBA-to-Python migration is complete (12 of 12 macros), and the web application is in active development. All core validation logic has been migrated, enhanced, and tested to ensure regulatory compliance with UK MiFIR and ESMA RTS 22 requirements.

Purpose & Regulatory Context

The system supports compliance with:

  • UK MiFIR (Markets in Financial Instruments Regulation) — Transaction reporting obligations for investment firms
  • ESMA RTS 22 — Technical standards specifying the content and format of transaction reports
  • FCA requirements — UK-specific reporting obligations to the Financial Conduct Authority

What the System Does

  1. Quarterly Accuracy Testing: Validates a sample of reported transactions against format and logic rules, generating corrections where errors are found
  2. SQL Extract Generation: Generates database queries to retrieve the transaction samples for testing
  3. Data Push: Pushes validated corrections back to master tracking files for submission to the Approved Reporting Mechanism (ARM)
  4. Quarterly Replay: Compares third-party upload files against accuracy testing results, correcting inconsistencies
  5. FIRDS Reportability: Determines whether financial instruments are reportable under MiFIR using FCA FIRDS data
  6. GLEIF LEI Lookup: Validates Legal Entity Identifiers and retrieves entity information from GLEIF Golden Copy data
