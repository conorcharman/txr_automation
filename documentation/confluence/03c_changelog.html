Phase 9 — Fullstack Web Application (April 2026)

Status: In Progress

Added

  • FastAPI backend (api/) — REST API with 13 routers: health, dashboard, jobs, configs, accuracy, replay, firds, gleif, utilities, filesystem, scheduler, pipelines, reconciliations
  • Pydantic v2 request/response schemas with strict validation and camelCase aliasing for JSON serialisation
  • SQLAlchemy 2.0 async ORM models for Job, Schedule, Pipeline, ReconciliationSchedule, and SavedConfig with async PostgreSQL backend
  • Celery task queue with Redis broker — async execution of all validation scripts, SQL generation, data push, replay processing, FIRDS/GLEIF cache operations, and format conversions
  • Celery Beat periodic task scheduler — triggers schedules, pipelines, and reconciliation runs automatically
  • WebSocket log streaming via Redis pub/sub — real-time task output per job (WS /api/ws/jobs/{job_id}/logs)
  • React 19 single-page application (web/) with TypeScript 6, Vite 8, and Tailwind CSS 4 — 10 pages: Dashboard, Accuracy Testing, Replay, FIRDS, GLEIF, Utilities, Scheduler, Reconciliation, Jobs, Job Detail
  • shadcn/ui component library with Radix UI primitives
  • Zustand 5 for client-side state management
  • TanStack React Query 5 for server state and cache management with 30-second stale time
  • React Hook Form 7 + Zod 4 for form validation across all script parameter forms
  • Saved Configurations feature — persist and reload script parameter sets via GET/POST/PUT/DELETE /api/configs
  • Pipeline scheduler — define ordered sequences of accuracy testing scripts with shared fiscal year, quarter, and per-stage path overrides via /api/pipelines
  • Reconciliation scheduler — schedule periodic runs of selected validation scripts for ongoing reconciliation via /api/reconciliations
  • Synchronous FIRDS reportability lookup (GET /api/firds/lookup?isin=&date=) — instant result without a background job
  • Synchronous GLEIF LEI lookup (GET /api/gleif/lookup?lei=) and entity name search (GET /api/gleif/search?name=)
  • Server-side filesystem browser (GET /api/filesystem/browse) used by the frontend path-picker component; restricted to /app/data, /app/config, and /app/src
  • Incident auto-discovery (POST /api/accuracy/run-incidents) — scan a directory to discover extract files and run selected incidents in one request
  • Docker Compose deployment with 6 services: Redis 7, PostgreSQL 16, API (Uvicorn), Celery Worker, Celery Beat, Web (Nginx)
  • Production Docker Compose variant (docker-compose.prod.yml) with optimised builds, memory limits, and health checks on all services
  • Nginx reverse proxy configuration with separate location blocks for WebSocket (/api/ws/) and REST (/api/) with SPA fallback to index.html
  • OpenAPI documentation disabled in production (/docs and /redoc only available in dev environment)
  • Production safety validation in Settings.validate_production() — rejects placeholder SECRET_KEY and database credentials
  • Script registry of 27 scripts in api/routers/jobs.py mapping identifiers to Python module paths for Celery dispatch

Phase 6 — Integration & Testing (March 2026)

Status: In Progress

Added

  • FIRDS module (src/firds/) — FCA FIRDS instrument reportability checking with SQLite + FTS5 cache and REST API client
  • GLEIF module (src/gleif/) — GLEIF LEI lookup service with SQLite cache, full-text search, and golden copy downloads
  • GUI application (src/gui/) — PySide6 desktop app with 6 tabs: Accuracy, Scheduler, Replay, FIRDS, GLEIF, Utilities
  • Scheduler engine (src/gui/scheduler/) — cron-based schedule queue, pipeline executor, and QSettings persistence
  • System tray application (src/gui/tray/) — background service with Windows toast notifications and auto-start on login
  • Automation CLI (txr-automate) — headless automation entry point for scheduled/unattended runs
  • Period extract generator (generate-period-extract) — period-based SQL template and DTF generation for System i Data Transfer
  • Incorrect net amount validation (validate-incorrect-net-amount) — replaces legacy pricing validation script
  • Replay XLSX converter (replay-xlsx-converter) — converts replay outputs to XLSX format for stakeholder distribution
  • Config validation tool (scripts/validate_config_migration.py) — verifies v1→v2 config migration
  • Merge Inconsistent IDs script (merge_inconsistent_ids.py) for deduplicating summary files
  • Performance benchmarking scripts in scripts/
  • Comprehensive documentation suite (Confluence HTML format, 15+ pages)

Changed

  • ESMA regex pattern corrections: NL (9 chars, not 11), ES (9 chars, not 10), GB patterns updated to match ESMA CID guidance
  • Inconsistent ID detection now uses prefix-aware nationality grouping — eliminates false positives from nationality changes
  • Configuration system upgraded to v2.0 with batch/single mode support and environment variable cascade
  • Deprecated v1 configuration templates removed; replaced with v2 branch-specific templates
  • Error messages now include expected format lengths calculated from regex patterns
  • JNT (joint account) false positive correction — improved detection logic
  • CONCAT surname handling enhanced for multi-part surnames with prefix removal
  • Fallback ID detection fix — corrected pattern matching for Person Code used as ID

Fixed

  • Dutch (NL) NIDN regex was accepting 11-character values instead of the correct 9
  • Spanish (ES) NIDN regex was accepting 10-character values instead of the correct 9
  • UK NINO regex updated to reject temporary NINOs (prefix TN)
  • JNT account pairs were generating false positive inconsistencies
  • Correction logic v2 typo fix (fix_correction_typo.py)

Remaining

  • test_batch_validation.py needs updating to v2.0 config format (~8 test configurations)

Phase 0 — Refactoring (January 2026)

Status: Complete

Added

  • Shared core library (src/core/) extracted from existing replay processing scripts
  • StructuredLogger — JSON-structured logging with file + console handlers, context injection
  • ConfigManager — YAML configuration with environment variable cascade (TXR_* prefix)
  • DateParser — multi-format date parsing (25+ recognised formats)
  • FileDiscovery — auto-discovery of input files in configured directories
  • safe_open_csv utility for encoding-safe CSV operations
  • pytest framework with fixtures and conftest.py
  • 35 initial test cases

Changed

  • Replay Phase 2 processor refactored to v4.2 — now uses core library for config, logging, and data structures
  • Replay Phase 3 processor refactored to v5.2 — added hash table indexing for O(1) transaction reference lookups
  • All existing scripts migrated from ad-hoc logging to StructuredLogger
  • All existing scripts migrated from hardcoded paths to YAML configuration
