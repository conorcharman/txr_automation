Accuracy Testing Commands

validate-buyer

Entry point accuracy_testing.scripts.buyer_id_validation:main
Purpose Validate buyer identification codes (NIDN, CONCAT, CCPT) for incidents 7_35, 7_37, 7_39
Modes Single / Batch

validate-buyer --config config.yaml --dry-run --progress
validate-buyer --use-env --log-level DEBUG

Common Arguments

Most CLI scripts share these common arguments:

Argument Description Default
--config PATH Path to YAML configuration file Auto-discovers config/local/...
--use-env Use TXR_* environment variables instead of YAML file false
--log-level LEVEL Override logging level (DEBUG, INFO, WARNING, ERROR) INFO
--dry-run Preview validation without writing output files false
--progress Show progress bar during processing false
--verbose Enable verbose console output false
--help Show help text and available arguments —
