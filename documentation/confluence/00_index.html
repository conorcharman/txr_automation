Document Conventions

  • British English is used throughout (e.g., “colour”, “organisation”, “behaviour”)
  • Code references use monospace formatting
  • File paths are relative to the project root (txr_automation/)
  • Client-specific data has been generalised throughout — no client names or confidential data appear in this documentation
